# Storyboard (Script-First)

Before writing any code, you MUST extract the narrative from the source content and map it to scenes.

## THE #1 RULE: The video IS the script

The video must tell the EXACT SAME STORY as the source content. Every scene's text must be derived directly from the source post/script. Do NOT paraphrase, reinterpret, or invent new messaging. The video is a visual amplification of the original words, not a remix.

## Script-First Workflow (MANDATORY)

Before writing ANY Remotion code, complete this extraction step:

**Step A: Extract the narrative beats**

Read the source content line by line. Identify every distinct beat/idea. Write them out as a numbered list with the EXACT words or a tight condensation (max 6-8 words per title, 15-20 per subtitle). Example:

```
Source post:
"R.I.P SaaS. AI agents just made half your tech stack obsolete."
"That $2,400/month software stack? An AI agent can replace 80% of it."
"I watched a founder rip out 11 tools. Replaced them with 4 agents."
"His CRM auto-updates itself now..."

Extracted beats:
1. HOOK: "R.I.P SaaS." / "AI agents made your tech stack obsolete"
2. PROBLEM: "That $2,400/mo stack?" / show 11 tool cards
3. SHIFT: "11 tools" struck through -> "4 agents"
4. BREAKDOWN: "CRM auto-updates itself" / "Sequences write themselves"
5. NUMBERS: "$2,400/mo" -> "$200/mo in API calls"
6. INSIGHT: "You don't need a platform." / "You need a system that thinks."
7. CTA: "Agents as a Service" / comment keyword
```

**Step B: Map beats to scenes**

Each beat = one scene. Assign duration, visual treatment, and transition. This is your storyboard.

**Step C: Cross-check before coding**

Before writing code, verify:
- Every scene title/subtitle uses words FROM the source content (not invented alternatives)
- The narrative arc matches: hook -> problem -> shift -> proof -> insight -> CTA
- No scene contradicts or deviates from the source message
- The order of ideas matches the source flow

## Scene Structure

Each scene needs:
- The EXACT text from the source (condensed to 6-8 words for titles, 15-20 for subtitles)
- A duration (3-5s per scene)
- The visual treatment (background, colors, animation style)
- The transition to the next scene

Typical structure for a LinkedIn-to-video conversion:
1. **Hook scene** (3-4s) -- The post's first 1-2 lines, verbatim or tightly condensed
2. **Problem/setup scenes** (5s each) -- The pain point as described in the post
3. **Solution/shift scene** (5s) -- The turning point, using the post's exact framing
4. **Proof/details scenes** (5s each) -- Data and specifics from the post
5. **Insight/quote scene** (5s) -- The post's quotable line, verbatim
6. **CTA scene** (5s) -- The post's call to action or closing message

## LinkedIn Post to Video Mapping

1. **Lines 1-2** (the hook) -> Scene 1: Big bold text, dramatic entrance
2. **The problem/pain** -> Scene 2: Visualize the pain (cards, numbers, red accents)
3. **The turning point** -> Scene 3: Visual transformation (strikethrough, morph, color shift)
4. **The proof/details** -> Scene 4-5: Lists get staggered cards, numbers get counters, comparisons get before/after
5. **The quotable insight** -> Scene 6: Typewriter or slow reveal, softer colors
6. **The CTA** -> Scene 7: Gold/amber accent, pulsing button, keyword prompt

## Key Principles

- **THE VIDEO IS THE SCRIPT** -- every visible text string must trace back to the source content
- **One idea per scene** -- never cram two concepts into one scene
- **Text must be SHORT** -- max 6-8 words for titles, 15-20 for subtitles
- **Visual hierarchy** -- biggest text is the key message, smaller text is context
- **Pacing** -- alternate between high-energy scenes (bouncy springs, bold colors) and calm scenes (smooth fades, muted tones)
- **Mobile-first** -- portrait format, large text, generous spacing
- **The hook scene must be DRAMATIC** -- this is the thumbnail and first impression
- **Never contradict the source** -- if the post says "4 agents replaced 11 tools", don't show agents being replaced by something else
